üõ† Why {purrr}? Link to heading

Iteration in R can be clunky using for-loops, especially when working with lists and complex data structures. {purrr} provides elegant functional programming tools that make iteration cleaner, safer, and more expressive.

üîπ Why Use {purrr}? Link to heading

‚úîÔ∏è Replaces slow and verbose for-loops
‚úîÔ∏è Works seamlessly with lists and tibbles
‚úîÔ∏è Encourages a functional programming approach
‚úîÔ∏è Improves error handling in iterative tasks
‚úîÔ∏è Simplifies nested operations

Let’s explore the key functions that make {purrr} a game-changer.

üìö Key {purrr} Functions Link to heading

Function Purpose
map() Apply a function to each element of a list
map_dbl(), map_chr(), map_lgl() Ensure output as numeric, character, or logical vectors
map2() Iterate over two lists at once
pmap() Iterate over multiple lists simultaneously
possibly(), safely() Handle errors gracefully
reduce() Perform iterative reduction (e.g., cumulative sums, merging)
walk() Apply functions for side effects (e.g., printing, logging)

üìä Example 1: Applying a Function to a List of Values Link to heading

Imagine we need to take the square root of a list of numbers, handling potential errors.

‚û°Ô∏è Base R approach using a for-loop: Link to heading 

numbers <- list(4, 9, 16, 25, -1)
results <- c()

for (x in numbers) {
  results <- c(results, sqrt(x))
}

‚úîÔ∏è Works, but not scalable for more complex cases.

‚û°Ô∏è {purrr} approach using map(): Link to heading 

library(purrr)
numbers <- list(4, 9, 16, 25, -1)
results <- map(numbers, sqrt)

‚úÖ More concise and readable
‚úÖ Handles list structures effortlessly
‚úÖ Easily extendable to more complex functions

üìä Example 2: Handling Errors with possibly() Link to heading

Some operations might fail. {purrr} helps us handle errors gracefully using possibly().

‚û°Ô∏è Handling errors in a safe way: Link to heading 

safe_sqrt <- possibly(sqrt, otherwise = NA)
results <- map(numbers, safe_sqrt)

‚úÖ Prevents the entire operation from failing due to a single error

üìä Example 3: Iterating Over Two Lists with map2() Link to heading

We can iterate over two lists simultaneously.

‚û°Ô∏è Multiply corresponding elements in two lists: Link to heading 

list1 <- list(2, 4, 6)
list2 <- list(3, 5, 7)

map2(list1, list2, ~ .x * .y)

‚úÖ Efficiently processes paired elements from two lists.

üìä Example 4: Working with Data Frames Using pmap() Link to heading

When we need to apply a function across multiple columns in a dataframe, pmap() is the best choice.

‚û°Ô∏è Example: Compute a BMI column Link to heading 

library(dplyr)
df <- tibble(Weight = c(70, 80, 65), Height = c(1.75, 1.82, 1.68))

df <- df %>%
  mutate(BMI = pmap_dbl(list(Weight, Height), ~ .x / (.y^2)))

‚úÖ Computes BMI for each row efficiently.

üìä Example 5: Reducing a List Using reduce() Link to heading

If we need to perform cumulative operations, reduce() is our go-to function.

‚û°Ô∏è Compute the cumulative sum of a list: Link to heading 

values <- list(1, 2, 3, 4, 5)
sum_result <- reduce(values, `+`)

‚úÖ Returns the total sum of all values in the list.

üìä Example 6: Using walk() for Side Effects Link to heading

walk() is similar to map() but is used when the function has side effects (e.g., printing, saving files, logging messages).

‚û°Ô∏è Print each element in a list: Link to heading 

walk(numbers, print)

‚úÖ Runs functions without returning output, useful for logging or debugging.

üìà Complete Workflow: Functional Programming with {purrr} Link to heading

Let’s put everything together into a cohesive pipeline.

library(purrr)
library(dplyr)

# Sample dataset
df <- tibble(
  Name = c("Alice", "Bob", "Charlie"),
  Score1 = c(85, 90, 78),
  Score2 = c(80, 95, 88)
)

# Apply functions using purrr
df <- df %>%
  mutate(
    Avg_Score = map2_dbl(Score1, Score2, ~ mean(c(.x, .y))),
    Score_Summary = pmap_chr(list(Name, Score1, Score2), 
                             ~ paste(.x, "has an average score of", mean(c(.y, .z))))
  )

‚úÖ Uses map2() and pmap() to efficiently compute new variables
